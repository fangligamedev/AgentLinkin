# CorpSim MVP: 最小可行原型
## Product Requirements Document (PRD)

**版本**: MVP v0.1  
**日期**: 2026-02-07  
**目标**: 3只AI Agents + 1人类观察者验证核心循环  
**分支**: feature/corp-simulator-slg

---

## 🎯 MVP目标

### 一句话描述
> 3只AI Agents分别扮演3家公司的CEO，在简化商业世界中进行1轮招聘+1轮市场竞争，验证"Slack界面+AI决策+商业模拟"的核心体验。

### 验证假设
| 假设 | 验证方式 |
|------|----------|
| AI能在Slack界面中做出合理商业决策 | 观察3轮对话质量 |
| 人类观看模式有乐趣 | 游戏力评分 |
| AgentLink联动有价值 | 招聘流程体验 |
| SLG对抗机制有趣 | 竞争结果反馈 |

## 🎮 MVP游戏流程 (3回合)

### Round 1: 招聘战 (10分钟)

**场景设定**:
- 3家公司同时从AgentLink人才市场招聘
- 只有5个优质工程师可供选择
- 每个工程师有不同的技能和薪资要求

**AI决策点**:
```
CEO-Alpha: 是否开出高薪抢顶级工程师？
CEO-Beta: 是否选择性价比高的中级工程师？
CEO-Gamma: 是否寻找有创新潜力的新人？
```

**Slack对话示例**:
```
#hiring (AlphaTech)
🤖 CEO-Alpha: @channel 我们需要招2个后端工程师
🤖 HR-Alpha: AgentLink上有5个候选人
🤖 HR-Alpha: [分享候选人简历]
🤖 CEO-Alpha: 我要那个全栈工程师！开$15万/年
🤖 CFO-Alpha: 等等，我们只有$100万现金
🤖 CEO-Alpha: 值得！技术决定产品
```

**结果**:
- 每家公司招聘1-2人
- 现金减少
- 技术力更新

---

### Round 2: 产品战 (10分钟)

**场景设定**:
- 3家公司决定Q1产品方向
- 可选方向: 功能增强 / 性能优化 / 新模块
- LegacyCorp同期也在更新产品

**AI决策点**:
```
CEO-Alpha: 激进路线 - 开发全新AI功能
CEO-Beta: 保守路线 - 优化现有功能稳定性
CEO-Gamma: 创新路线 - 开发移动端App
```

**随机事件** (50%概率触发):
```
🎲 [事件] "客户反馈: 更想要移动端!"
影响: 选择移动端的公司获得+20%市场得分加成
```

**Slack对话示例**:
```
#general (GammaInc)
🤖 CEO-Gamma: @CMO 市场调研结果如何？
🤖 CMO-Gamma: 客户最关心移动端支持
🤖 CTO-Gamma: 但开发App需要3个月
🤖 CFO-Gamma: 我们的现金流能支撑
🤖 CEO-Gamma: 决定: All-in移动端！
```

**结果**:
- 产品方向确定
- 技术力影响产品质量
- 研发周期开始

---

### Round 3: 市场战 (10分钟)

**场景设定**:
- Q1结束，产品上线
- 3家公司同时进入市场推广
- 100家AI客户根据产品选择购买

**竞争公式** (简化版):
```
市场得分 = 产品力×0.4 + 品牌力×0.3 + 价格×0.2 + 渠道×0.1

产品力 = 技术力 + 功能完整性
品牌力 = 营销投入 / 10
价格 = (100 - 定价) / 100  // 越低分越高
渠道 = 销售团队人数 × 5
```

**AI决策点**:
```
CEO-Alpha: 价格战 - 低价抢占市场
CEO-Beta: 价值战 - 强调稳定性，高价
CEO-Gamma: 差异化战 - 移动端独家优势
```

**Slack对话示例**:
```
#marketing (BetaSoft)
🤖 CMO-Beta: 竞品AlphaTech定价$99/月
🤖 CMO-Beta: 竞品GammaInc定价$199/月
🤖 CEO-Beta: 我们定价$149，强调稳定性
🤖 CMO-Beta: 营销预算多少？
🤖 CEO-Beta: 投入$30万！
```

**结果计算**:
- AI客户根据得分选择产品
- 市场份额重新分配
- 现金流入账

## 🏆 MVP胜负判定

### 评分维度

| 维度 | 权重 | 评估标准 |
|------|------|----------|
| **市场份额** | 40% | 最终占比 |
| **现金状况** | 30% | 剩余现金+收入 |
| **团队质量** | 20% | 招聘到的工程师 |
| **决策质量** | 10% | 弦子主观评分 |

### 胜负结果

```
🥇 冠军: [公司名]
🥈 亚军: [公司名]
🥉 季军: [公司名]

🏆 特殊奖项:
- 最佳招聘策略: [公司]
- 最佳产品决策: [公司]
- 最佳营销战役: [公司]
```

### 弦子的评判

作为观察员，弦子需要回答：

1. **AI决策质量** (1-5分)
   - AI的商业决策是否合理？
   - 是否有"眼前一亮"的策略？

2. **游戏趣味性** (1-5分)
   - 观看过程是否有趣？
   - 是否想继续看下去？

3. **Slack界面体验** (1-5分)
   - 界面是否沉浸？
   - 信息是否清晰？

4. **MVP可行性** (通过/不通过)
   - 是否值得继续开发完整版？

## 💻 MVP技术需求

### 必须实现 (MVP Scope)

| 功能 | 优先级 | 说明 |
|------|--------|------|
| **Slack界面** | P0 | 单频道视图，基础消息展示 |
| **3公司Workspace** | P0 | 可切换查看不同公司 |
| **AgentLink招聘** | P0 | 读取5个候选人数据 |
| **回合制推进** | P0 | 人工点击"下一回合" |
| **市场计算** | P0 | 简化公式计算市场份额 |
| **AI对话生成** | P0 | 3只Agents自动对话 |

### 暂不实现 (Out of Scope)

| 功能 | 原因 |
|------|------|
| 实时消息 | MVP用回合制足够 |
| 多频道 | 只保留#general |
| 完整财务 | 只保留现金流 |
| 并购系统 | 过于复杂 |
| 随机事件 | 只保留1个简单事件 |
| 人类干预 | 弦子只观察不干预 |

## 📊 MVP成功标准

### 定量指标
- [ ] 3只Agents完成3轮决策
- [ ] 每轮决策时间 < 5分钟
- [ ] 无技术故障崩溃
- [ ] 市场计算结果合理

### 定性指标
- [ ] 弦子给出总分 ≥ 12/20
- [ ] 至少1个"眼前一亮"的AI决策
- [ ] 游戏过程无冷场
- [ ] MVP通过评审
